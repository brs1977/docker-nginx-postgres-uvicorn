Для организации API сервера и клиента необходимо проставить наименования полей в api и в разметке страницы.
В API заменить = ? на конкретное имя 
В разметке проставить data-имя для тегов в тексте которых будет стоять значение парамерта из api
Для каждой уникальной рабочей области определить тип

Список endpoint api
1. /page/{код_страницы} – получение структуры страницы по коды, 
Вход
	код_страницы - необязательный параметр, если не задан вернуть значения для страницы по умолчанию
Выход(json)
{
	код страницы = ?
	заголовок страницы = ?
	меню = ?, упорядоченный список элементов меню: [ 
		код меню = ?
		наименование пункта меню = ?
		тип меню = menu_type?, подменю, ссылка на страницы, сообщение
    код родителя меню = ?
    текст сообщения = ?
    заголовок сообщения = ?
		код страницы = ?
		признак активности меню = ?
	]
	пользатель = ?, до авторизациии пусто(null) {
		фио = ?
		статус = ?
		дата время начала работы = ?, в формате ISO
	},
	помощь = ?, вкладка в рабочей области, список строк [
		строка помощи = ?
	]
	рабочая область = ? {
		заголовок рабочей области = ?
		тип рабочей области = ?
		фон рабочей области = ?
		иконка рабочей области = ?
		содержание рабочей области = ?, разное для типа 
	}
}
Пример содержимого для главной рабочей области
	n_par: [1],
	m_par: [{txt: 'Ссылка1','pic': 'иконка1'}]

2. /login - авторизация по cookie
Вход(json)
	username - пользователь
	password - пароль
Выход(201, без тела)
